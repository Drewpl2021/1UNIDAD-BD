📋 ¿Qué contiene el dataset?

Archivo Excel con transacciones de ventas de un negocio real (distribución de bebidas). Cada fila representa un ítem dentro de una factura. Una misma factura puede tener múltiples ítems.

📑 Facturas y boletas 🛒 Ítems por comprobante 👥 Datos de clientes 💰 Montos en soles (PEN) 📅 Fechas de emisión 📦 Descripción de productos
📖 Diccionario de datos (columnas reales)
Columna original Normalizada Tipo Descripción
FECHA DE EMISIÓN fecha_de_emision Date Fecha de emisión del comprobante (dd/MM/yyyy → DateType)
TIPO tipo Integer Tipo de comprobante (3=boleta, 1=factura)
SERIE serie String Serie del comprobante (ej: BQQ1, FQQ1)
NÚMERO numero Integer Número correlativo del comprobante
DOC ENTIDAD NÚMERO doc_entidad_numero String Número de documento del cliente (clave de cliente)
DENOMINACIÓN ENTIDAD denominacion_entidad String Nombre o razón social del cliente
MONEDA moneda String Moneda de la transacción (PEN, USD)
CÓDIGO codigo String Código interno del producto
DESCRIPCIÓN descripcion String Nombre del producto/servicio (ej: PEPSI 2000 ML - PQT)
CANTIDAD cantidad Double Unidades vendidas del ítem
SUBTOTAL subtotal Double Monto sin IGV de la línea
IGV igv Double Impuesto General a las Ventas (18%)
TOTAL total Double Monto total pagado por la línea (con IGV)
ANULADO anulado String Indica si el comprobante fue anulado (SI/NO)
🔍 Problemas de calidad detectados y resueltos
  • ✓
    Columnas con caracteres especiales — tildes y espacios. Resuelto: normalización con unicodedata.
  • ✓
    Fechas como texto — formato dd/MM/yyyy. Resuelto: to_date(col, "dd/MM/yyyy").
  • ✓
    Registros anulados — comprobantes con ANULADO="SI". Resuelto: filtro aplicado.
  • ✓
    Totales ≤ 0 — líneas con TOTAL nulo o negativo. Resuelto: filter(TOTAL > 0).
  • ✓
    Tabla de clientes sin fuente externa — generada con dropDuplicates del mismo dataset.

⚡ Lazy Evaluation — transformaciones vs acciones
Operación Tipo ¿Spark ejecuta? Ejemplo
.select() TRANSFORMACIÓN ❌ No — plan lógico df.select("total")
.filter() TRANSFORMACIÓN ❌ No — agrega filtro al DAG df.filter(col("total") > 0)
.withColumn() TRANSFORMACIÓN ❌ No — agrega columna al plan df.withColumn("anio", year(...))
.show() ACCIÓN ✅ Sí — ejecuta el plan df_clean.show(5)
.count() ACCIÓN ✅ Sí — ejecuta el plan df_clean.count()
.write.parquet() ACCIÓN ✅ Sí — escribe en disco df.write.parquet(path)

💻 Código clave — Broadcast Join + Función Ventana
NB03 · Broadcast Join
# Tabla de clientes únicos (dimensión extraída del mismo dataset)
clientes = ventas_limpio.select("doc_entidad_numero", "denominacion_entidad").dropDuplicates()

# Broadcast Join — clientes pequeño, se envía a todos los nodos
df_join = ventas_limpio.join(broadcast(clientes), "doc_entidad_numero", "inner")

# Función Ventana: ranking y total acumulado por cliente
ws = Window.partitionBy("doc_entidad_numero").orderBy("FECHA_DT")
df_window = df_join
.withColumn("ranking", row_number().over(ws))
.withColumn("total_acumulado", _sum("total").over(ws))
